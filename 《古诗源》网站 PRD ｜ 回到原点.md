# 《古诗源》网站 PRD ｜ 回到原点

一个把唐以前的诗放回它诞生位置的个人阅读站。诗歌传统的源头，加上汉字本身的源头，两条线叠在一起，是这个站区别于任何一个古诗大全的地方。当作个人作品慢慢打磨，做深比做全重要。

## 定位

《古诗源》是清代沈德潜编的诗选，专收唐以前（先秦、汉、魏、晋、南北朝、隋）的诗。它的立意是：唐诗那座高峰的水，是从这些更早、更朴的诗里流出来的。这个站承接这个立意，做两件市面上的古诗站不做的事。

一是让诗回到诗歌传统的源头，把《诗经》、汉乐府、古诗十九首这些放在它们本来的位置，让读者看见后世名句从哪里流出来。二是让字回到汉字的源头，这些早期诗用字少、直、重，离文字刚被造出来时的生命力最近，值得把关键字的字形演变摆出来。

这个站的价值在纯和深，不在全。

## 目标用户与场景

第一读者是我自己。一个安静读古诗、顺手查字源、慢慢往里加内容的地方。次要读者是少数同好，能读进去、也在意排版和字的人。

典型场景：睡前打开，一屏一首诗，读到一个字有感觉，点开看它从甲骨文到今天怎么长出来的，再顺着线索翻到后世谁化用了这一句。整个过程安静、不被打断。

## 设计哲学：回到原点

把克制本身做成产品的气质。

- 做减法。没有推荐流、没有相关阅读、没有弹窗，一屏尽量只承载一首诗。
- 排版讲究到底。大留白，中文用 LXGW WenKai 这类温润的楷宋，竖排作为可选阅读模式。打开像翻一本好书，不像刷一个网站。
- 信息密度低，但每一处都经得起看。宁可少收，也不堆砌注释和资料。

## 内容结构

内容按《古诗源》原书的朝代分卷组织：先秦、汉、魏、晋、宋齐梁陈（南北朝）、隋。每一卷下是诗人，诗人下是诗。

每首诗是一个内容单元，包含这几层，从必备到可选：

- 诗题、作者、朝代、原文（必备）
- 挑选出的承重字及其字形演变数据（P0 签名功能依赖这一层）
- 源流线索：这首诗或其中某句被后世谁化用、脱胎（P1）
- 简注、白话、朗读（P2，后置，可长期留空）

数据上把诗和字拆成结构化内容，字形演变单独建一份字库，多首诗可以复用同一个字的数据。

## 功能优先级

### P0：先把这三样做透

**字的原点（签名功能）。** 每首诗挑几个承重字，点开看它从甲骨文、金文、小篆到今天的字形演变，配一句这个字最早画的是什么、本义是什么。这是别人最难抄、也最能立住这个站的功能。第一版可以只手工做几十首诗里的高频字，字形图和释义来源用汉典、象形字典这类公开字源资料，逐步扩充。

**克制的阅读体验。** 一屏一诗的阅读页，大留白，横排为主、竖排可切换，LXGW WenKai 中文字体。这是地基，字的原点功能要挂在一个足够安静的界面上才成立。

**基础内容与浏览。** 按朝代分卷浏览，能从卷到诗人到诗一路点进去。第一版内容不求全，先放进《诗经》选、古诗十九首、汉乐府、曹操等几组最能代表源头的诗。

### P1：让源流看得见

**源流线索。** 选一些人人会背的唐宋名句，标出它化用或脱胎自《古诗源》里的哪一句，双向可跳转。曹操的月明星稀往后接多少人写月，古诗十九首的生命感慨往后接李白杜甫。让读者亲眼看见源和流，比任何介绍都有说服力。

**站内检索。** 按诗题、作者、字检索。等内容多到翻不过来时再做，早期没必要。

### P2：后置，可长期不做

简注、白话翻译、朗读音频、个人收藏与批注。这些是加分项，堆早了会稀释这个站的纯度。等核心三样打磨到满意，再按兴趣一样一样加。

## 技术选型建议

本地优先、静态站的思路最合适一个慢慢打磨的个人作品。

- 前端框架用 Next.js，走 SSG（静态生成）。诗和字都是相对稳定的内容，构建期生成静态页，加载快、SEO 好、部署便宜。
- 内容用结构化数据管理，诗一份、字库一份，用 Markdown 加 frontmatter 或本地 JSON，方便你一个人手工增补，也方便诗和字互相引用。
- 字形演变第一版可以直接用图片资源，后面有余力再考虑矢量字形或字体渲染。
- 部署放 cloudflare平台，改完推一下就上线，符合慢慢打磨、随时改的节奏。

### 样式方案

样式分两层，各管各的。

- 页面骨架、布局、通用间距用 Tailwind，写得快，档位预设让整站天然协调。要用 shadcn 组件也顺，它本来就依赖 Tailwind。
- 阅读页的中文排版是这个站的命根子（字体、行高、字间距、竖排、标点悬挂这些），用自己写的 CSS 死磕，不交给框架预设。Tailwind 的默认档位帮不上这种一像素一像素抠的字面功夫。

### 组件库

交互零件按需从 shadcn/ui 取，复制源码进项目，只拿用得上的那几个，不整包引入。

- 字形演变弹层用 Popover 或 Dialog（焦点管理、Esc 关闭、点外面关闭都处理好了）
- 主题切换、竖排横排切换用 Toggle 或 Switch
- P1 站内检索用 Command
- 源流线索提示用 Tooltip

背景记一笔，免得按旧信息选型。截至 2026-07，shadcn 新项目默认底层已从 Radix 换成 Base UI，官方也推荐新项目用 Base UI。Radix 没有被废弃，仍然支持，初始化加 `-b radix` 可切回。Base UI 是 Radix 和 MUI 部分核心成员合流后的新一代 headless 库，包更小、在积极迭代，对一个追求克制的站是加分。

## 打磨路线

当作长期作品，按能立刻看到成果的顺序推进，每一步都是一个能自己用起来的完整版本。

1. 立骨架：阅读页排版加朝代浏览，放进先秦到汉的几组核心诗，界面先安静下来。
2. 上签名功能：给已收录的诗做字的原点，先覆盖高频承重字，这一步做完这个站就有了别人没有的东西。
3. 补源流：给最有代表性的十几首诗接上后世源流线索。
4. 扩内容：按卷把《古诗源》的诗逐步补齐，字库同步扩充。
5. 按兴趣加 P2：检索、简注、朗读、收藏，想做哪个做哪个。

## 明确不做的事

- 不做古诗词大全，不追求收全所有朝代所有诗。范围就锁在《古诗源》这本书里。
- 不做社交、评论、打卡、积分这类留存功能。
- 不做花哨动效和信息流，克制是这个站的底线。
- 前期不做账号体系，个人作品用不上。
